# Phase 매핑 제안 (승인 대기) — 2026-07-20

이 문서는 docs/plans/2026-07-20-1st_Fut_list.md 기반 제안이며, 사용자 승인 전까지 어떤 phase_XX.md도
수정하지 않는다. Phase 2(투두+메모 위젯)는 진행 중이므로 이 제안에서 어떤 항목도 추가하지 않았다.

대상 범위: tag가 MUST 또는 SHOULD인 항목 전체(85건). 그중 alreadyDecided=false인 항목은 이번 제안에서
Phase를 새로 설계했고, alreadyDecided=true인 항목은 이미 확정된 Phase를 그대로 기록만 했다(재배정 없음).
DEFER/SKIP 태그 항목은 이 표에 전혀 포함하지 않았다 — docs/FEATURES.md 백로그에만 남는다.

워크로드 상한 검증: 총 85건 중 가장 큰 Phase는 Phase 7(17건, 20.0%)이며, 어떤 Phase도 전체의 50%를
넘지 않는다.

## 1. 신규 Phase 제안

기존 14 Phase 로드맵 어디에도 대응하지 않는 두 대분류(H 뉴스 브리핑, I 픽셀 오리 오피스)에 대해 3개의
신규 Phase를 제안한다.

### Phase 15 — 뉴스 브리핑 파이프라인 (수집·요약·발행)

- **근거**: RSS 피드 수집, Gemini 3줄 요약, 중복 기사 클러스터링, 오리 아침 브리핑 연계, 데일리
  뉴스레터 자기 발송(Gmail API), 피드 관리 UI, 죽은 피드 감지, 발송 실패 처리까지 뉴스 브리핑
  파이프라인 전체(H 카테고리, 13건)가 기존 14 Phase 로드맵 어디에도 대응하지 않는다. RSS·공개 API만
  사용하고 본문 크롤링·전문 재게시를 금지하는 저작권 제약이 파이프라인 전반에 걸려 있어 하나의
  Phase로 묶어 설계·구현하는 것이 일관성이 있다.
- **배정 항목 수**: 13건 (전체의 15.3%)

### Phase 16 — 픽셀 오리 오피스 기반 (이벤트·렌더링)

- **근거**: 원문 "[구현 상세 설계 지침]" 순서(사전조사 → 이벤트소스 → 중계 → 렌더링 → 스프라이트 →
  상태매핑 → 역할배치 → 클릭 상호작용)를 따라, 플레이어가 직접 조작하지 않아도 완결되는 시각화
  인프라를 먼저 구축하는 Phase다. 유사 오픈소스(Pixel Agents류) 사전 조사를 첫 Task로 지정한다.
  Tauri sidecar 로컬 WebSocket 보안(J6), 이벤트 스로틀링, 픽셀 오피스 리소스 예산, 퇴근 모드(빈 화면
  방지) 같은 인프라 안전장치도 함께 묶는다.
- **배정 항목 수**: 12건 (I 카테고리 11건 + J 카테고리 1건, 전체의 14.1%)

### Phase 17 — 픽셀 오리 오피스 상호작용 (플레이어 조작)

- **근거**: 대장오리 플레이어 캐릭터의 그리드 이동·충돌 판정·카메라 팔로우, 직원 오리 상호작용(읽기),
  에이전트 수에 따른 동적 오피스 레이아웃은 모두 Phase 16의 렌더링 인프라가 먼저 서 있어야 검증
  가능한 후속 기능이다. Phase 15/16과 합칠 경우 워크로드가 과도하게 커지지 않도록(50% 룰) 별도
  Phase로 분리했다.
- **배정 항목 수**: 3건 (전체의 3.5%)

## 3. DEFER / SKIP 항목 처리

tag가 DEFER 또는 SKIP인 모든 항목(alreadyDecided 값과 무관)은 이번 로드맵 배정 대상에서 완전히
제외했다. 이들은 phase_XX.md에 반영하지 않으며, docs/FEATURES.md 백로그에만 남아 있는다. 백로그
항목이 향후 승격될 경우 별도의 사고 게이트(재검토)를 거쳐야 한다.
