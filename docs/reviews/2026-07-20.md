## 실제 버그 (lessons-learned 후보)

- **POST 라우트의 기본 리다이렉트가 메서드를 보존해 405 발생** (logout, proxy 미인증 리다이렉트 2곳).
  `NextResponse.redirect()` 기본 상태코드 307이 원 요청 메서드(POST)를 유지 -> GET만 받는 페이지 라우트에서 405.
  실사용자 클릭 테스트로 발견, 303으로 수정(커밋 0909813, ac73174) 후 프로덕션에서 재검증 완료.
  상세: docs/anti-patterns/post-redirect-get.md, 기록: docs/lessons-learned.md
